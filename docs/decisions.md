# 決定ログ (Decision Log)

> プロダクト / UX / アーキテクチャの「**決めたこと**」を記録する**単一の正**。**追記専用**。
> 一度決めたことを**蒸し返さない**ための装置。会話で決まった重要事項は、その場でここへ落とす。
>
> **運用ルール**
> - 重要な決定をしたら、**実装と同じコミットで**ここに 1 項目追記する（後回しにしない）。
> - 既存の決定を**覆す/変える**ときは、古い項目を消さず `状態: Superseded by Dxx` にし、新項目を足す。
> - 「決定そのもの」をここに。詳細は各ドキュメントへ（視覚値=`design-principles.md`/`status-card-design.md`、
>   用語=`glossary.md`、背景=`concept.md`、ロードマップ=`navi-roadmap.md`）。
> - 着手前の**未決の論点**はここではなく `issues/` に。決まった瞬間にここへ昇格する。
>
> 記法: `## Dn. タイトル` ／ 日付・状態 ／ **決定** ／ 理由 ／ 却下した代替 ／ 関連。

---

## D1. ステータスカードは自動消滅させない（手動 dismiss まで残す）

- 日付: 2026-06-26 ／ 状態: **Accepted**
- **決定**: ステータスカードは時間経過やセッション終了で**自動的に消さない**。ユーザーが明示的に
  dismiss するまで残す。特に**アテンション**（`waiting_approval` / `error`）は、当該セッションが
  次の状態へ進んでも勝手に消えてはならない。
- **理由**: navi の中核価値は「見逃さない」こと。通知が自動で消えると、席を外している間に
  `waiting_approval` / `error` を取りこぼし、常駐通知の意味が無くなる。
- **却下した代替**: 「Live Activity 的にセッション終了で自動消滅」。純度は高いが**見逃しリスク**が
  決定的に不可。`duration` で自動消滅するのは**ペット発話 (PetSpeech)** のみ（別物・glossary §1）。
- **帰結**: 手動 dismiss の所作が UI 上必須 → 「閉じ方」を独自設計する必要がある（`issues/` で継続）。
- **関連**: `glossary.md`（ステータスカード / Live Activity 行）、`design-principles.md` P2。

## D2. app は desktop のみ対象。モバイル考慮は playground 限定

- 日付: 2026-06-26 ／ 状態: **Accepted**
- **決定**: プロダクト（Tauri デスクトップ常駐アプリ）の UI は**デスクトップのみ**を対象とする。
  モバイルでの見え方・操作は、ブラウザ公開される **playground (`examples/playground`) だけ**の関心事。
- **理由**: navi はデスクトップ常駐アプリ。playground は設計サンドボックス（たまたまブラウザ＝モバイルで開ける）。
- **帰結**: app の操作設計では **hover を一級の手段として使ってよい**（例: hover で閉じる）。
  スワイプ/長押し等のタッチ前提を app の必須要件にしない（playground での体験向上は任意）。
- **関連**: `concept.md`、`frontend-packaging.md`（app と playground の分離）。

## D3. state の一次表現は右上のステータスアイコン

- 日付: 2026-06-26 ／ 状態: **Accepted**
- **決定**: セッションの state は、カード**右上のステータスアイコン**を一次表現とする
  （本家 codex pets の右上アイコンを踏襲）。色 accent（左レール等）は**周辺視の補助**に降格し、
  色で前に出すのは**アテンション**（`waiting_approval` / `error`）に限る。
- **理由**: 色だけの表現は誤解が多い。アイコン＝精密、色＝周辺視、の二段が最も誤解が少ない
  （`design-principles.md` P3 形が先・色は意味のために / P4 光学）。
- **未決（→ issues）**: 右上をステータスに使うため空く「閉じる」操作の具体 → **D4 で決定**。
  state→アイコンの図像セット自体は `issues/f4a1b2` で継続。
- **関連**: `status-card-design.md`（state→アイコン対応・accent）、`glossary.md`（アテンション）、D4。

## D4. 閉じるは左上の丸いアイコンボタン（ソースバッジと morph・hover で出現）

- 日付: 2026-06-26 ／ 状態: **Accepted**
- **決定**: ステータスカードの「閉じる」は、**左上の丸いアイコンボタン**とする（本家 codex pets の踏襲）。
  既定は**ソースバッジ**を表示し、**カード hover で crossfade して✖（削除）に切り替わる**（同位置・同じ丸形で morph）。
  常設の閉じるボタンは置かない。右上はステータスアイコン専用（D3）に明け渡す。
- **理由**: D3 で右上をステータスに使うと閉じる場所が空く。D2（desktop 専用）で hover を一級に使えるため、
  本家流の hover 出現が成立。左右の角を 1 つずつの丸い要素に整理でき、静けさ（`design-principles.md` P2）を保てる。
- **却下した代替**: 「右上ステータスの左隣に✖を増設」（要素が増える）／「ステータスアイコン自体が✖に変化」
  （状態が隠れる・右上を二役にすると D3 と衝突）。
- **帰結 / 既知のトレードオフ**: hover 中はソースバッジ（出自）が一時的に隠れる。タイトルに source 名が出るため許容。
- **関連**: D2・D3、`status-card-design.md`（close 仕様）、`issues/f4a1b2`。
